# Эмулятозавр

Эмулятор ретро-компьютеров.

Для установки необходимо иметь GIT и Node.js.


В первой версии реализован эмулятор Микроши.

Эмулятор работает в окне браузера. Для его старта нужно запустить сервер и клиент. Позже напишу скрипт для старта.
А пока нужно в отдельных консолях:
- сервер: cd server;  node index.js
- клиент: cd client; npm start
- в браузере ввести URL: http://localhost:3001/

В текущем варианте эмулятор совмещён с отладчиком. Даже если пользователю это не нужно 😁. Поэтому для начала работы нужно нажать кнопку отладчика Play (►►)

## Микроша

Как пользоваться Микрошей - описано в документации (data\МИКРОША\microsha-official-guide.pdf). 
Но эмуляторы клавиатуры и магнитофона требуют отдельных пояснений:

### Клавиатура

Имеет три режима:
- **Mouse**: Дефолтный режим. Клик на виртуальной клавише (кроме УС и НР) эмулирует клик на Микроше. Залипание не поддерживается. Для УС и НР клик переключает между нажатым и отжатым состоянием. Основной недостаток: если надо ввести более 5 символов, то это уже утомляет.
- **Debug**: Имеет смысл только в режиме отладки. Для всех виртуальных клавиш клик переключает между нажатым и отжатым состоянием.
- **Keyboard**: Позволяет использовать реальную клавиатуру для эмуляции. Но клавиатура Микроши сильно отличается от стандартной раскладки. Поэтому не получилась везде удачно совместить верхний и нижний регистры. (Особенно печально со скобками). 
А ещё в этом режиме в ряде случаев не получится использовать клавиатуру по прямому назначению для других фичей приложения. Например, перестаёт работать ввод в выпадающих окошках. В общем, тоже не идеальный вариант.

Конечно, было бы хорошо чтобы можно было вводить просто символы на клавиатуре, а они бы напрямую передавались в эмулятор. 
Но для этого недостаточно эмулировать контроллер клавиатуры. Потому что символы из портов контроллера формируются системным монитором Микроши по довольно сложным правилам. Поэтому, пока это сделать не получилось.

### Магнитофон

Есть две разные папки: data и userData. Из первой предполагается только чтение. Там лежат стандартные приложения, которые поставлялись на кассетах для Микроши. А userData используется чтобы писать и читать пользовательские данные. Например, программы на Бейсике.

Эмулятор магнитофона поддерживает два режима:
- **Напрямую** - это прямой обмен данными между компьютером и памятью эмулятора. Этот режим рекомендуется, чтобы загружать приложения (например, Бейсик). У них есть стандартный формат: в заголовке указан адрес начальной загрузки, а в конце - контрольная сумма. После загрузки выводится сообщение, по какому адресу произошла загрузка. Обычно 0. И для старта приложения нужно использовать директиву G0000. Где 0000 - это начальный адрес. А если это действительно 0, то можно просто G. Данный режим заменяет директивы системного монитора I и O. Но некоторые приложения (н.р. Бейсик) имеют свои функции для чтения/записи. И для них этот режим не подойдёт.
- **Эмуляция** - эмулируется работа контроллера. Для системного монитора это директивы I и O, хотя это намного медленнее, чем прямой режим. Однако, если чтением и записью управляет приложение, то поможет только **Эмуляция**. При этом, есть довольно мало времени между "включением" магнитофона и вводом соответствующей команды. Этот процесс описан в документации в разделе взаимодействия с магнитофоном. Например, если в Бейсике нужно сохранить текущую программу, то сначала нужно ввести MSAVE (но не нажимать ВК), затем нажать кнопку "Начать запись на магнитофон" (↥), ввести имя файла (без расширения rkm), выбрать режим **Эмуляция**, нажать Start и ВК на виртуально клавиатуре.
